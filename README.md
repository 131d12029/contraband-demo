## 系統概述

### 平台定位
「違禁商品通報平台」是一個政府民眾互動型的電商違規舉報系統，旨在協助社會大眾便捷地通報網路平台上販售的政府公告禁售商品，並提供透明的案件追蹤與統計分析功能。

### 核心目標
- **便捷通報**：簡化用戶舉報流程，鼓勵民眾參與監督
- **透明追蹤**：用戶可查詢自己通報案件的處理進度
- **數據洞察**：提供政府部門實時統計分析數據
- **跨平台覆蓋**：支援多家主要電商平台（蝦皮、momo、PChome、露天、Yahoo、樂天）

### 用戶群體
- **主要用戶**：社會大眾、消費者、網購用戶
- **次要用戶**：政府監督部門、稽查機構、平台管理員

### 架構特點

1. **單頁應用 (SPA)** 
   - 所有內容在一個 HTML 文件中
   - 無需後端服務器即可獨立運行
   - 基於頁面切換的虛擬路由

2. **模塊化組織**
   - CSS 變量系統統一設計語言
   - JavaScript 按功能分組
   - 清晰的命名規範

3. **本地優先設計**
   - 優先使用 LocalStorage 存儲用戶提交數據
   - 基礎數據集內置，無需 API 調用
   - 安全隱私考量

4. **前端完全驅動**
   - 所有邏輯在客戶端執行
   - 無後端依賴
   - 高穩定性、快速加載

### 模塊 2：表單與通報流程

**位置**：立即通報頁 (Report Page)

**設計理念**：
- **分步表單** (Stepped Form)：2步驟流程，降低用戶認知負荷
- **實時反饋**：URL 自動平台偵測、必填欄位驗證
- **進度指示**：視覺化步驟指示器

**Step 1 - 商品與平台資訊**

字段配置：

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 商品頁面網址 | URL Input | ✓ | 觸發自動平台偵測 |
| 電商平台 | Select Dropdown | ✓ | 8個主要平台選項 |
| 賣家名稱/ID | Text Input | ✗ | 可選，增強案件追蹤精度 |
| 商品名稱 | Text Input | ✓ | 通報核心信息 |
| 禁售類別 | Select Dropdown | ✓ | 9類預定義違規類別 |
| 佐證截圖 | File Upload | ✗ | 最多3張，支援JPG/PNG |
| 補充說明 | Textarea | ✗ | 允許詳細描述 |
| 通報人Email | Email Input | ✗ | 接收進度通知 |

**Step 2 - 確認與送出**

內容：
- 顯示 Step 1 所有已填信息的確認摘要
- 隱私聲明（依《個人資料保護法》處理）
- 「返回修改」和「確認送出通報」按鈕

**成功頁面**
- 顯示生成的案號（格式：`RPT-YYYY-NNNN`）
- 複製按鈕
- 查看紀錄的快速連結

**關鍵函數**：
```javascript
function goStep2()          // 驗證 → 顯示確認
function goStep1()          // 返回修改
function submitReport()     // 提交並生成案號
function detectPlatform()   // 自動偵測URL所屬平台
function previewFiles()     // 圖片預覽與刪除
```

**驗證規則**：
```javascript
必填驗證清單：
✓ 商品網址 (URL格式)
✓ 電商平台 (已選擇)
✓ 商品名稱 (非空)
✓ 禁售類別 (已選擇)
```

## 數據流程

### 通報流程 (Report Flow)

```

    ↓
自動偵測平台 (detectPlatform)
    ↓ (URL 識別成功)
"偵測到平台" 綠色提示出現，select自動填充
    ↓
用戶填寫 Step 1 其他內容
    ↓
點擊「下一步」
    ↓
驗證必填字段 (goStep2)
    ↓ (驗證通過)
進入 Step 2，顯示確認摘要
    ↓
點擊「確認送出通報」
    ↓
submitReport() 執行：
  1. 收集所有表單數據
  2. 生成案號 (RPT-YYYY-NNNN)
  3. 創建紀錄對象
  4. 調用 saveRecord() → LocalStorage
  5. 顯示成功頁面
    ↓
用戶可選擇：複製案號 / 查看紀錄
```

### 數據存儲流程

```
BASE_RECORDS (內置20筆)
    ↓
loadRecords()
    ↓
LocalStorage 中查詢 'rpt_records'
    ↓ (如果存在)
[...userRecords, ...BASE_RECORDS]
    ↓
返回合併的 MOCK_RECORDS
    ↓ (用於顯示和篩選)
```

### 篩選與渲染流程

```
用戶改變篩選條件
    ↓
filterRecords() 觸發
    ↓
讀取 4 個篩選器的值
    ↓
MOCK_RECORDS.filter() - 應用所有條件
    ↓
renderRecords(filtered) - 動態生成 HTML
    ↓
更新記錄計數
    ↓
用戶可查看和展開各記錄
```

### 統計計算流程

```
顯示統計頁面
    ↓
updateStats() 執行
    ↓
計算：
  - total     = MOCK_RECORDS.length
  - done      = filter(status==='done').length
  - reviewing = filter(status==='reviewing').length
  - rate      = (done/total)*100%
    ↓
initCharts() 執行
    ↓
讀取預設的模擬數據 + 動態統計數據
    ↓
Chart.js 初始化 4 個圖表
    ↓
渲染完成
```
